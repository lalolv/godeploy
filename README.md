# 自动部署服务

- [x] 支持多个项目部署

## 使用说明

- 执行路径：{域名}:{端口}/deploy/{执行脚本名称}。执行脚本名称中不带有.sh 扩展名。
- 在 shells 目录下面，保存对应的脚本文件名（扩展名为 xxx.sh）。

例如：访问 POST `http://127.0.0.1:8080/deploy/demo`，执行 `shells/demo.sh` 脚本。
